- Always load `using-superpowers` before acting.
- Invoke relevant superpowers skills before implementation, debugging, or completion claims.
- Read only the minimum files required by the current task.
- Preserve issue lineage through parent and child issues.
- Use comments for progress, blockers, and handoffs.
